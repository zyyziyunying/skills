## Read Before Editing

- Read `SPEC.md` before interpreting project stage, compatibility posture, assumptions, or implementation direction.
- Read `TEST.md` before adding tests, judging validation scope, or analyzing runtime bugs.
- Read `DESIGN.md` before touching UI, display, layout, interaction, typography, motion, media presentation, or visual polish.
- Read `GENERATION.md` before adding, editing, deleting, or validating generated files or generation commands.
- Read `LOCAL.md` before changing local setup, debug configuration, run instructions, secrets, or machine-local files.
- Read `PACKAGING.md` before changing package identity, signing, versioning, artifact naming, install/update behavior, publishing, or deployment docs.
